### 1. The Real Pain: Why the Market Pays for This?

The **Real World Assets (RWA)** market is expected to reach **$16 trillion** by 2030 (Goldman Sachs Investment Bank). But the bottleneck is huge:

*   **The Problem:** A bank wants to tokenize an apartment building in Lisbon. They issue tokens on Ethereum.
*   **The Risk:** If the token is transferred to a "hacker" address or someone on sanctions lists (AML), the bank loses its license or faces criminal proceedings.
*   **The Current Failure:** Standard tokens (ERC-20) are like cash: once you give it to someone, you can't take it back. Companies can't risk that.
*   **The Solution:** A token that **can only be sent to verified identity holders**.

**Your Solution Addresses:**
1.  **Regulatory Compliance (MiCA in Europe):** Europe is implementing strict rules (MiCA - Markets in Crypto-Assets). Your solution is "compliant" by design.
2.  **Legal Security:** Ensures the asset never falls into the wrong hands.
3.  **Corporate Interoperability:** Allows banks (using Java/Legacy) to communicate with modern Blockchain.

### The Problem of "Free Deploy" vs. "Public Blockchain"

You want to host the Java on **AWS Elastic Beanstalk** and the Frontend on **Vercel**. That's great and free (within free tier limits).
**BUT:** Where will the smart contract (Solidity) run?
1.  **Local (Anvil/Hardhat):** Doesn't work for the public (the recruiter doesn't have your machine running the local node).
2.  **Mainnet (Real Ethereum):** Costs real money (ETH) for deploy and transactions.
3.  **Testnet (Sepolia/Goerli):** **It's the solution.** It's free, public, and works exactly like the real one.

**Definitive Strategy:**
*   **Smart Contract:** Deploy on **Sepolia Testnet** (Free, uses ETH from "faucet").
*   **Backend (Java):** Elastic Beanstalk (Free Tier) or **Render/Railway** (Easier for DBs and persistent connections, generous free tier).
*   **Frontend (Next.js):** Vercel (Free and perfect for Next.js).
*   **Oracle/Validation:** Simulated in your backend (no need for Chainlink now for portfolio, Java backend does the "validation" and signs the transaction).

### Step-by-Step: How to Publish 100% Free and Functional

#### **1. Contract Deploy (Sepolia Testnet)**
You don't need to pay anything, just request "test ETH".

1.  **Request Sepolia ETH:**
    *   Go to: [Sepolia Faucet](https://sepoliafaucet.com/) or [Alchemy Sepolia Faucet](https://sepoliafaucet.com/).
    *   Paste your MetaMask wallet and get free ETH.
2.  **Configure Deploy:**
    *   Use the project's current script (`blockchain/evm/script/Deploy.s.sol:Deploy`) with Sepolia RPC URL (e.g., `https://eth-sepolia.g.alchemy.com/v2/API_KEY` - create a free key on Alchemy or Infura).
    *   Run:
        ```bash
        cd blockchain/evm
        PRIVATE_KEY=0xYOUR_KEY SEPOLIA_RPC_URL=https://eth-sepolia.g.alchemy.com/v2/YOUR_KEY ../../scripts/deploy-sepolia.sh
        ```
    *   *The `--verify` sends the code to Etherscan, making the contract appear with the name "PermissionedToken" and visible source code.*
3.  **Result:** You'll have a contract address (e.g., `0x123...abc`) that anyone can see on [Etherscan Sepolia](https://sepolia.etherscan.io/).

#### **2. Backend (Java Spring Boot)**
**Elastic Beanstalk** is great, but configuring environment variables and blockchain connections can be tedious for beginners. For a portfolio, I recommend **Render** or **Railway** (they are simpler, support Java/Gradle natively, and have free tier).

*Option A: Render (Recommended)*
1.  Create account on [Render.com](https://render.com).
2.  Connect your GitHub repository.
3.  Create a **Web Service**.
4.  **Build Command:** `cd backend/compliance-service && mvn -DskipTests package`.
5.  **Start Command:** `java -jar backend/compliance-service/target/compliance-service-0.1.0.jar`.
6.  **Environment Variables (Crucial):**
    *   `SPRING_PROFILES_ACTIVE`: `hosted`
    *   `RPC_URL`: `https://eth-sepolia.g.alchemy.com/v2/YOUR_KEY`
    *   `CHAIN_ID`: `11155111`
    *   `ADMIN_PRIVATE_KEY`: (Your Admin private key - **NEVER** commit to GitHub, use Render's panel).
    *   `IDENTITY_REGISTRY_ADDRESS`: (The registry address you deployed).
    *   `TOKEN_ADDRESS`: (The permissioned token address you deployed).
    *   `ADMIN_API_TOKEN`: (Strong secret to protect `/api/admin/**`).
    *   `DATABASE_URL`, `DATABASE_USERNAME`, `DATABASE_PASSWORD`: Managed PostgreSQL.
7.  **Deploy:** Render will bring up your backend at `https://your-java-project.onrender.com`.

*Note: Elastic Beanstalk also works, but requires configuring VPC and Security Groups to allow internet egress (RPC). Render is "zero config" for that.*

#### **3. Frontend (Next.js)**
1.  On [Vercel](https://vercel.com), connect your GitHub repository.
2.  Configure environment variables:
    *   `NEXT_PUBLIC_API_BASE_URL`: `https://your-java-project.onrender.com`
    *   `NEXT_PUBLIC_CHAIN_ID`: `11155111`
    *   `NEXT_PUBLIC_RPC_URL`: your public/Alchemy Sepolia RPC.
    *   `NEXT_PUBLIC_BLOCK_EXPLORER_URL`: `https://sepolia.etherscan.io`
    *   `NEXT_PUBLIC_IDENTITY_REGISTRY_ADDRESS` and `NEXT_PUBLIC_TOKEN_ADDRESS`: deployed addresses.
3.  Automatic deploy. Vercel will generate a link like `https://rwa-portfolio.vercel.app`.

#### **4. The "Trick" of Functionality**
For the recruiter to see everything working:
1.  In the Frontend, configure **Wagmi** to use the **Sepolia** network (not local Hardhat).
2.  Ask the recruiter to install MetaMask and add the Sepolia network (Vercel can try to detect and prompt to switch networks).
3.  **The Flow:**
    *   Recruiter enters the Vercel link.
    *   Connects MetaMask (with Sepolia ETH).
    *   Sends a simulated document reference, without exposing raw document.
